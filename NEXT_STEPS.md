# Reserve-One 次のステップ

ログインができたら、以下の流れで使い始められます。

---

## 1. 管理者として枠を作成する

1. **Supabase** の `supabase_schema.sql` 内で、`admin@example.com` を自分のメールアドレスに変更して SQL を実行済みか確認
2. そのメールで**新規登録**すると自動で管理者になります
3. **管理画面**（/admin）で「枠を作成する」から日付と時刻を入力して枠を追加
   - 例: 日付 `2025-02-03`、時刻 `10:00, 11:00, 12:00, 14:00`

---

## 2. カレンダーで予約する

- **顧客**（/dashboard）で週カレンダーを表示
- **空き**枠の「予約」ボタンで予約
- **予約済み**の枠は「解除」でキャンセル可能
- 週の左右ボタンで前後の週に移動

---

## 3. デプロイして Web で公開する

1. **GitHub** に push（[SETUP.md](./SETUP.md) の「2. コードを GitHub に連携」参照）
2. **Vercel** でプロジェクトをインポート
3. **Environment Variables** に Supabase の URL と anon key を設定
4. デプロイ完了後、本番 URL でアクセス

---

## 4. パスワードについて

Supabase Auth の「メール確認」がオフの場合、未確認でもログインできます。本番では **Supabase ダッシュボード → Authentication → Providers → Email** で「Confirm email」をオンにし、確認メールを有効にすることを推奨します。
